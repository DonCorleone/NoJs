• animation-timeline: scroll() — links animation progress directly to the page scroll position
  • animation-timeline replaces animation-duration — time is driven by scroll, not a clock
  • transform: scaleX() + transform-origin: left — scales from the left edge, not from center
01
Scroll down to see the progress bar fill. It stays pinned to the top of the viewport and grows as you move through the page — no JavaScript involved.
02
The animation-timeline: scroll() property links the keyframe animation directly to the document's scroll position. No scroll event listeners. No requestAnimationFrame.
03
Because it's a CSS animation, it's hardware-accelerated. The browser handles the compositing — smooth, jank-free, and off the main thread.
04
The keyframe scales the bar horizontally: from { transform: scaleX(0) } to to { transform: scaleX(1) }. Animating transform instead of width keeps it GPU-accelerated and glitch-free.
05
You can attach animation-timeline: scroll() to any element and any property — opacity, transform, color. The progress bar is just the most familiar use case.
06
Browser support is around 73% as of 2025 — Chrome and Edge are fully on board, Firefox too. Safari is the main holdout, so consider it a progressive enhancement for now.
